# Desafio Técnico – SRM Asset
## Contexto
A SRM Asset é uma gestora de recursos brasileira, fundada em 2005, líder no mercado de crédito estruturado e pioneira em Fundos de Investimento em Direitos Creditórios (FIDCs). Especializada em antecipação de recebíveis (middle corporate) e trade finance, a empresa atua com foco em multicedentes e multisacados, movimentando mais de R$ 10 bilhões anualmente e atendendo milhares de clientes no Brasil. 

Este desafio técnico avalia suas habilidades em desenvolvimento e arquitetura.

## Perfis e Níveis

### Back-end:

* Júnior: Criar uma API simples de clientes e solicitações.

* Pleno: Adicionar lógica de análise de crédito (ex.: aprovar ou rejeitar).

* Sênior: Arquitetar eventos assíncronos, justificar decisões de infraestrutura.

### Front-end:

* Júnior: Consumir uma API simples (clientes e solicitações) e exibir.

* Pleno: Consumir API com análise de crédito e histórico.

* Sênior: Consumir API paginada, com filtros, e apresentar soluções complexas.

### Full-stack:

* Júnior: Construir API simples e interface básica.

* Pleno: API com análise de crédito e interface completa.

* Sênior: Sistema completo, com decisões de ponta a ponta.

## Contratos (Front-end)

* Júnior: Consumir o JSON [básico](/junior.json) (clientes e solicitações simples).

* Pleno: Consumir JSON com [análise de crédito e histórico](/pleno.json).

* Sênior: Consumir JSON com [paginação e filtros](/senior.json).
